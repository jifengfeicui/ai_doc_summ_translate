# 前端 OCR 功能使用说明

## 功能概述

前端已添加OCR开关选项，用户可以在创建任务时选择是否启用OCR模式来处理扫描版PDF或图片型PDF文档。

## 修改的文件

### 1. `ui/src/api/index.js`
更新了API调用函数，添加`useOcr`参数：
- `createTask(fileId, taskType, useOcr = false)`
- `createTasksBatch(fileIds, taskType, useOcr = false)`

### 2. `ui/src/views/FilesView.vue`
批量任务创建页面添加OCR开关：
- 在页面头部添加了"启用OCR"开关
- 添加了帮助提示图标，说明OCR功能的用途
- 批量创建任务时会将OCR选项传递给后端
- 成功提示中会显示是否启用了OCR

### 3. `ui/src/views/FileDetailView.vue`
单文件详情页面添加OCR开关：
- 在"创建任务"区域添加了OCR选项
- 使用灰色背景块突出显示OCR设置
- 添加了帮助提示图标
- 创建任务时会将OCR选项传递给后端
- 成功提示中会显示是否启用了OCR

## 使用步骤

### 批量创建任务
1. 在文件列表页面选择要处理的文件（勾选复选框）
2. 如需使用OCR模式，打开"启用OCR"开关
3. 点击"批量总结"或"批量翻译"按钮
4. 系统会创建任务并显示成功消息（包含OCR状态提示）

### 单个任务创建
1. 进入文件详情页面
2. 在"创建任务"区域，如需使用OCR模式，打开"启用OCR"开关
3. 点击"创建总结任务"或"创建翻译任务"
4. 系统会创建任务并显示成功消息（包含OCR状态提示）

## 功能说明

### OCR开关
- **默认状态**: 关闭（false）
- **适用场景**: 扫描版PDF、图片型PDF文档
- **提示信息**: "启用OCR模式处理扫描版PDF或图片型PDF，处理时间会较长"

### 视觉设计
- 使用 Element Plus 的 `el-switch` 组件
- 开关文字：
  - 打开时显示"启用OCR"
  - 关闭时显示"关闭OCR"
- 帮助图标使用 `QuestionFilled` 图标，鼠标悬停显示详细说明
- 单文件页面的OCR选项使用浅灰色背景突出显示

## 技术细节

### 数据流
```
用户操作 → 前端开关状态(useOcr) → API调用 → 后端处理
```

### 状态管理
- 使用 Vue 3 Composition API 的 `ref` 管理开关状态
- 默认值为 `false`
- 两个页面的开关状态独立管理

### API 参数
```javascript
// 单个任务
{
  task_type: "summarize" | "translate",
  use_ocr: true | false
}

// 批量任务
{
  file_ids: [1, 2, 3],
  task_type: "summarize" | "translate",
  use_ocr: true | false
}
```

## 用户提示

### 成功消息示例
- 不使用OCR: "总结任务创建成功"
- 使用OCR: "总结任务创建成功 (启用OCR)"
- 批量不使用OCR: "批量总结任务创建完成：成功创建 3 个任务"
- 批量使用OCR: "批量总结任务创建完成 (启用OCR)：成功创建 3 个任务"

## 注意事项

1. OCR模式会显著增加处理时间，前端会在帮助提示中说明
2. OCR选项对所有任务类型（总结、翻译）都有效
3. 每次创建任务时可以独立选择是否启用OCR
4. OCR状态不会被持久化保存，每次都需要重新选择

## 兼容性

- 需要 Element Plus 组件库支持
- 需要 `@element-plus/icons-vue` 图标库
- 支持所有现代浏览器
- 与 Electron 桌面应用完全兼容
